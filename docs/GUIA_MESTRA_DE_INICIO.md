# Guia Mestre de Início

Este é o documento que deve ser lido primeiro sempre que eu iniciar um trabalho neste repositório.

## Objetivo

Reduzir a quantidade de documentos abertos no começo e manter uma ordem única de leitura, execução e validação.

## Ordem de leitura

1. Ler este guia mestre.
2. Identificar se a tarefa é de backend, frontend, documentação ou validação.
3. Abrir apenas o documento complementar necessário para a tarefa.
4. Fazer a menor mudança possível.
5. Validar com o comando mais barato que realmente confirma a alteração.

## Documentos que podem ser unificados aqui

- [README_START.md](README_START.md) e [COMO_RODAR.md](COMO_RODAR.md): entrada de execução, portas, inicialização e troubleshooting básico.
- [GUIA_CODIGO_BACK_FRONT.md](GUIA_CODIGO_BACK_FRONT.md), [GUIA_CLASSE_METODO_V2.md](GUIA_CLASSE_METODO_V2.md), [MAPA_VISUAL_DO_SISTEMA.md](MAPA_VISUAL_DO_SISTEMA.md), [FLUXO_GUIADO_MEDICO.md](FLUXO_GUIADO_MEDICO.md) e [CHECKLIST_INTERATIVO_MEDICO.md](CHECKLIST_INTERATIVO_MEDICO.md): navegação de arquitetura e fluxos operacionais.
- [TESTING_GUIDE.md](TESTING_GUIDE.md) e [CHECKLIST_DEBUG_RAPIDO.md](CHECKLIST_DEBUG_RAPIDO.md): validação e diagnóstico.
- [INDICE_DOCUMENTOS.md](INDICE_DOCUMENTOS.md): catálogo geral, que pode ser enxugado depois para apontar só para este guia e para os documentos realmente especiais.

## O que continua separado

- Relatórios de auditoria e histórico técnico.
- Conteúdo legado em [arquivo_legacy/](arquivo_legacy/).
- Qualquer documento que exista só para registrar uma decisão pontual ou um problema antigo.

## Protocolo padrão de início

- Confirmar o contexto da tarefa.
- Ler este guia antes de escolher outros documentos.
- Se a tarefa tocar o backend, abrir o guia de fluxo ou a auditoria técnica apenas se necessário.
- Se a tarefa tocar a inicialização local, usar primeiro este guia e depois o documento de execução.
- Se a tarefa pedir correção, começar pelo arquivo mais próximo do comportamento, não por documentação ampla.

## Regra de uso

Este guia deve ser o ponto de partida padrão. Os outros documentos viram referências especializadas, não entrada principal.
